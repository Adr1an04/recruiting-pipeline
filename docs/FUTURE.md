# Roadmap

Erga is pre-alpha. This roadmap communicates direction, not guaranteed dates or compatibility.

## Near term

- Improve first-run guidance and configuration diagnostics.
- Add explicit review and approval flows for imported evidence candidates.
- Strengthen application-history reporting and mail-event review.
- Expand synthetic fixtures for ATS pages and résumé templates.
- Improve packaging and reproducible release artifacts.

## Later

- Support additional reviewable résumé formats without weakening provenance.
- Add opt-in semantic retrieval as a supplement to deterministic matching.
- Improve local research summaries while keeping source citations distinct from verified evidence.
- Add migration tooling for configuration and local schemas before stable releases.
- Explore a local graphical interface over the same domain layer.

## Explicit non-goals

- automatic job application submission;
- autonomous recruiter messaging;
- fabricated résumé claims or inferred metrics;
- broad mailbox permissions;
- silent mutation of a source résumé or remote Overleaf project; and
- making Hermes, MCP, or a hosted service mandatory for the core workflow.

Open a feature request when proposing roadmap work. Explain the user problem, the smallest useful
scope, and how the proposal preserves Erga's privacy and approval boundaries.
